## Mudancas tecnicas (economia de servidor e bugs)

### 1. Remocao do `window.location.reload()` apos sync

- **Arquivo:** `dash/components/DashboardClient.tsx`
- Substituido por marcacao dos anos selecionados como `partial=true` no estado `partialYears`. Refaz fetch apenas dos anos afetados sem descartar estado React.

### 2. Cache-Control na rota de activities

- **Arquivo:** `dash/app/api/activities/route.ts`
- Header `Cache-Control: private, max-age=120, stale-while-revalidate=30`. Browser cacheia 2 minutos; navegacao entre anos nao bate no Firestore enquanto cache e valido.

### 3. Field projection nas queries Firestore

- **Arquivo:** `dash/app/api/activities/route.ts`
- `.select(...)` listando apenas campos consumidos pelo frontend. Exclui `syncedAt`, `reviewNote`, `reviewedAt`, `reviewedBy`. Reduz custo de leitura.

### 4. Limite de historico por plano

- **Arquivos:** `dash/lib/access.ts`, `dash/lib/dashboard.ts`
- `FREE_PLAN_YEARS = 1`, `PRO_PLAN_YEARS = 3`, `CURRENT_YEAR`.
- `buildAllowedYears(yearsBack)` gera array de anos permitidos dinamicamente.
- free: 1 ano | pro: 3 anos | admin/master: tudo. Atualiza automaticamente em janeiro sem codigo.

### 5. Bug: reset de paginacao ao ignorar atividade

- **Arquivo:** `dash/components/DashboardClient.tsx`
- `setSelectedSports` sempre criava novo array mesmo com mesmos valores, disparando `setPage(1)`.
- Corrigido com guard: `if (valid.length === current.length) return current`.

### 6. Best efforts backfill no sync incremental

- **Arquivos:** `dash/app/api/strava/sync/route.ts`, `dash/lib/strava.ts`
- Apos batch de novas atividades, loop busca ate 24 corridas antigas sem `bestEfforts` e enriquece com fetch da API Strava.
- Escreve apenas o campo `{ bestEfforts }` com merge — nao sobrescreve o documento.

### 7. metadataRepairedAt no sync

- **Arquivo:** `dash/app/api/strava/sync/route.ts`
- Toda escrita de meta passa a incluir `metadataRepairedAt: new Date()`.
- Evita full scan a cada page load; a flag confirma que o meta ja foi validado.

## Saude: sono e peso (CSV Garmin)

### 8. Parsing de CSV Garmin

- **Arquivo novo:** `dash/lib/health-csv.ts`
- `parseHealthCSV(content)` detecta tipo pelo cabecalho.
- Sono: cabecalho `Sono 4 semanas,...` — gera registros diarios `{ date, durationMin, sleepTime, wakeTime }`.
- Peso: cabecalho `Tempo,Peso,...` — formato hierarquico com linhas de data e linhas de hora; guarda primeiro registro do dia.
- Fix critico: datas no CSV de peso vinham como `" 30 Jun 2026",` com virgula no fim. Corrigido com `.replace(/,+$/, '')` antes do regex.

### 9. API de upload (admin)

- **Arquivo novo:** `dash/app/api/admin/upload-health/route.ts`
- POST multipart com `files[]`. Chama `parseHealthCSV` por arquivo, batch-write no Firestore (`sleepRef` / `weightRef`).
- Retorna `{ sleepSaved, weightSaved, skipped, errors }`.

### 10. API de leitura de saude

- **Arquivo novo:** `dash/app/api/health/route.ts`
- GET admin-only com `?from=&to=`. Retorna `{ sleep: [], weight: [] }`.
- `Cache-Control: private, max-age=300`.

### 11. Colecoes Firestore

- **Arquivo:** `dash/lib/firebase.ts`
- Adicionado `sleepRef(stravaId)` e `weightRef(stravaId)` como subcollections do usuario.

### 12. Tipos

- **Arquivo:** `dash/components/dashboard/types.ts`
- Adicionado `SleepRecord` e `WeightRecord`.

### 13. Graficos de saude no dashboard
