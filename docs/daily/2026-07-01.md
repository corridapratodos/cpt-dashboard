## Saude: sono e peso (CSV Garmin)

### 8. Parsing de CSV Garmin

- **Arquivo novo:** `dash/lib/health-csv.ts`
- `parseHealthCSV(content)` detecta tipo pelo cabecalho.
- Sono: cabecalho `Sono 4 semanas,...` — gera registros diarios `{ date, durationMin, sleepTime, wakeTime }`.
- Peso: cabecalho `Tempo,Peso,...` — formato hierarquico com linhas de data e linhas de hora; guarda primeiro registro do dia.
- Fix critico: datas no CSV de peso vinham como `" 30 Jun 2026",` com virgula no fim. Corrigido com `.replace(/,+$/, '')` antes do regex.

### 9. API de upload (admin)

- **Arquivo novo:** `dash/app/api/admin/upload-health/route.ts`
- POST multipart com `files[]`. Chama `parseHealthCSV` por arquivo, batch-write no Firestore (`sleepRef` / `weightRef`).
- Retorna `{ sleepSaved, weightSaved, skipped, errors }`.

### 10. API de leitura de saude

### 11. Colecoes Firestore

### 29. Pre acesso com codigo de convite

- **Arquivos:** `dash/middleware.ts`, `dash/lib/pre-access.ts`, `dash/app/access/page.tsx`, `dash/app/api/access/grant/route.ts`, `dash/components/PreAccessGate.tsx`
- O dashboard agora pode ficar fechado por um codigo de convite antes do login Strava, sem exigir cadastro manual no Firestore.
- A liberacao gera um cookie `httpOnly` assinado no servidor e o middleware redireciona qualquer visitante sem esse cookie para `/access`.
- `PRE_ACCESS_CODES` controla a lista de codigos validos no ambiente. Se a variavel estiver vazia, a barreira fica desativada para nao quebrar ambientes locais ou staging rapido.
- `PRE_ACCESS_COOKIE_SECRET` pode ser configurado para assinar essa camada separadamente; se nao existir, o sistema reaproveita `NEXTAUTH_SECRET`.
