# Diario - 2026-07-04

## Contexto

- Inicio da rodada de otimizacao de leituras do Firestore para reduzir consumo diario mesmo com apenas um usuario ativo.

## Mudancas

### 1. Repair de metadata com cooldown

- **Arquivos:** `dash/app/page.tsx`
- A home continua capaz de reparar anos disponiveis quando o metadata estiver incompleto, mas deixa de tentar isso a cada request.
- Agora existe cooldown de 24h entre tentativas automaticas via `metadataRepairAttemptedAt`, reduzindo leituras pesadas acidentais no request path.

### 2. Full sync sem releitura da colecao inteira

- **Arquivos:** `dash/app/api/strava/sync/route.ts`
- O resumo do full sync passa a ser calculado a partir do conjunto que acabou de vir da API do Strava e foi persistido, em vez de reler toda a colecao no Firestore logo em seguida.
- Isso remove uma varredura completa adicional da base em cada full sync.

### 3. Backfill de best efforts por consulta dirigida

- **Arquivos:** `dash/app/api/admin/backfill-efforts/route.ts`
- O backfill deixa de carregar todas as atividades para filtrar em memoria.
- Agora ele busca apenas documentos com `bestEfforts = []`, ordenados por data e limitados por lote, reduzindo drasticamente as leituras por execucao.

### 4. Contratos de custo cobertos por smoke tests

- **Arquivos:** `dash/scripts/test-smoke.mjs`, `dash/tests/security-smoke.test.mjs`
- Os testes de fumaca agora validam a presenca do cooldown de repair e da consulta dirigida do backfill, ajudando a evitar regressao silenciosa de custo.

## Validacao

- `npm run test:smoke` aprovado.
- `npm run typecheck` aprovado.
