# Documentacao de acompanhamento

Este projeto mant?m um registro diario para controle tecnico e de produto.

## Estrutura

- `docs/daily/TEMPLATE.md`: modelo para novos registros.
- `docs/daily/YYYY-MM-DD.md`: registro do dia com mudancas, decisoes e proximos passos.

## Regra operacional

- Criar ou atualizar um documento diario a cada sessao de trabalho relevante.
- Registrar o que mudou no codigo, no setup e no comportamento do sistema.
- Anotar decisoes tecnicas, riscos, pendencias e proximos passos.
- Manter o texto curto e objetivo para facilitar auditoria e continuidade.
